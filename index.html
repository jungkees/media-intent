Media Over Intents (Pick Media Intent) Demo

Jungkee Song

This is a demo of W3C DAP WG's Pick Media Intent specification.

Making It Work

First, you need to be running a reasonably recent version of Chrome (I think 19+ will do). In a real version this would work with all browsers of course.

Then you need to download the extension that register the Intent. This demo provides two services, flickr search and YouTube search. In the real version this could be registered by the service page directly, but that isn't supported just yet.

Unpack the manifest, and follow these steps:

  1. Open the chrome://chrome/extensions page
  2. Check the “Developer mode” box if you haven't already
  3. Click “Load unpacked extension” and select the directory that contains the manifest in the dialog
  4. This should install an extension with an icon and all.

Once that's done, go to the client page (or you can write your own!). If you load media there, you should normally be prompted to select from registered Pick Media Intent services. Choosing that will open the service page, which in turn will allow you to search and pick some media objects.

Current Status of the Spec

Pick Media Intent is in FPWD and DAP WG is moving the spec toward LC. You can share your valuable feedback in DAP's public email list. Also, any reference service implementation is welcome to us!
